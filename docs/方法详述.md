# 方法（详细版）

本文档详细阐述本工作提出的**层次化音乐驱动舞蹈生成**方法。整体框架建立在
基于 Transformer 的扩散舞蹈生成模型之上，核心贡献是在原有"逐帧局部条件"
之外，显式引入**全局（整曲风格）→ 局部（逐帧细节）的层次化结构**，并配以
一个**时间段 × 身体部位**的层次化损失。

---

## 1. 问题定义与符号

给定一段音乐特征序列，模型学习生成对应的 3D 人体舞蹈动作。记：

- 动作张量 $x \in \mathbb{R}^{B \times T \times D}$，其中 $B$ 为 batch，
  $T=150$ 帧（30 fps，5 秒），$D=151$ 为动作通道。
- 音乐特征 $c \in \mathbb{R}^{B \times T \times F}$，逐帧的 Jukebox 特征，
  $F=4800$。
- 扩散时间步 $t \in \{1,\dots,N\}$，$N=1000$。

动作通道 $D=151$ 的布局为：

$$
x = [\underbrace{c_{1:4}}_{\text{脚触地}},\ \underbrace{r_{1:3}}_{\text{根平移}},\ \underbrace{q_{1:24\times 6}}_{\text{24 关节 6D 旋转}}]
$$

第 $j$ 个关节的旋转占据通道 $[7+6j,\ 13+6j)$。

**基线的局限**：原模型每一帧 $x_\tau$ 仅以该帧的局部音乐特征 $c_\tau$ 为条件，
没有任何对**整段音乐风格**的显式表征，导致全局编排的一致性完全依赖网络隐式
学习。

---

## 2. 扩散框架（预备）

采用 DDPM 前向加噪过程，对干净动作 $x_0$：

$$
q(x_t \mid x_0) = \mathcal{N}\!\big(x_t;\ \sqrt{\bar\alpha_t}\,x_0,\ (1-\bar\alpha_t)\,\mathbf{I}\big),
\qquad
x_t = \sqrt{\bar\alpha_t}\,x_0 + \sqrt{1-\bar\alpha_t}\,\epsilon,\ \ \epsilon\sim\mathcal{N}(0,\mathbf I)
$$

其中 $\bar\alpha_t = \prod_{s\le t}(1-\beta_s)$。去噪网络 $f_\theta(x_t, c, t)$
直接预测 $x_0$（即 `predict_epsilon=False` 设置下预测干净信号）。本方法的全部
改动都集中在条件如何进入 $f_\theta$ 以及损失如何加权，扩散调度本身不变。

---

## 3. 层次化条件：全局 → 局部

实现见 `model/model.py` 的 `DanceDecoder.forward`。

### 3.1 全局风格向量（第一层特性）

对整段音乐在时间维做**均值池化**，再经 MLP 编码，得到全局风格向量
$g \in \mathbb{R}^{B\times d}$（$d$ 为隐维）：

$$
\bar c = \frac{1}{T}\sum_{\tau=1}^{T} c_\tau \in \mathbb{R}^{B\times F},
\qquad
g = \mathrm{MLP}_{\text{global}}(\bar c)
$$

```python
self.global_encoder = nn.Sequential(
    nn.Linear(cond_feature_dim, latent_dim),  # F -> d
    nn.SiLU(),
    nn.Linear(latent_dim, latent_dim),        # d -> d
)
global_pool   = cond_embed.mean(dim=1)        # (B, F)
global_hidden = self.global_encoder(global_pool)  # g: (B, d)
```

$g$ 即**层次结构的第一层特性**：它刻画整曲的风格/能量，与任意单帧无关。

### 3.2 全局调制局部（FiLM）

将逐帧的局部条件 token $h^{\text{loc}}\in\mathbb{R}^{B\times T\times d}$ 用 $g$
做**特征级仿射调制**（FiLM）。由 $g$ 生成缩放 $\gamma$ 与平移 $\beta$：

$$
(\gamma, \beta) = \mathrm{DenseFiLM}(g),
\qquad
\tilde h^{\text{loc}}_\tau = (1+\gamma)\odot h^{\text{loc}}_\tau + \beta,\quad \forall \tau
$$

```python
cond_tokens = featurewise_affine(cond_tokens, self.global_film(global_hidden))
```

这是"全局引导局部"的核心：**同样的局部鼓点，在不同全局风格下被重新表达**。

### 3.3 双通道注入

除了调制局部 token，全局 token 还被注入解码器的另外两处，使网络能直接对其
注意：

**(a) 交叉注意力记忆**：将全局 token 拼接到注意力记忆序列尾部

$$
C = [\,\tilde h^{\text{loc}}\ ;\ h^{\text{time}}\ ;\ g\,] \in \mathbb{R}^{B\times(T+T'+1)\times d}
$$

**(b) FiLM 时间步条件**：把 $g$ 加到时间步条件向量 $t_{\text{cond}}$ 上

$$
t_{\text{cond}} \leftarrow t_{\text{cond}} + g
$$

```python
c = torch.cat((cond_tokens, t_tokens), dim=-2)
global_token = global_hidden.unsqueeze(1)
global_token = torch.where(keep_mask_embed, global_token, null_global_token)
c = torch.cat((c, global_token), dim=-2)                 # (a) 交叉注意力记忆
t = t + torch.where(keep_mask_hidden, global_hidden, 0)  # (b) FiLM 时间条件
```

随后 $f_\theta$ 的 Transformer 解码器以 $C$ 为记忆、$t_{\text{cond}}$ 为 FiLM
条件，对带噪动作 $x_t$ 去噪。

### 3.4 无分类器引导（CFG）

为支持无分类器引导，引入可学习的**空全局 token** $g_\varnothing$
（`null_global_token`）。训练时以概率 $p_{\text{drop}}$ 丢弃条件，此时用
$g_\varnothing$ 替换 $g$（与局部条件的丢弃同步）：

$$
g \leftarrow
\begin{cases}
g, & \text{保留条件}\\
g_\varnothing, & \text{丢弃条件（概率 } p_{\text{drop}}\text{）}
\end{cases}
$$

采样时按引导权重 $w$ 混合有/无条件预测：

$$
f_\theta^{\text{guided}} = f_\theta^{\varnothing} + w\,\big(f_\theta^{c} - f_\theta^{\varnothing}\big)
$$

```python
def guided_forward(self, x, cond_embed, times, guidance_weight):
    unc        = self.forward(x, cond_embed, times, cond_drop_prob=1)
    conditioned= self.forward(x, cond_embed, times, cond_drop_prob=0)
    return unc + (conditioned - unc) * guidance_weight
```

### 3.5 采样期全局约束 `global_pool_override`

提供一个钩子：采样时**直接指定全局风格**，而非从音乐计算。这正是局部
inpainting 无法表达的**第一层（全局）约束**：

$$
\bar c \leftarrow \bar c^{\text{ref}}\ \text{（来自另一段音乐/指定风格）},\qquad g = \mathrm{MLP}_{\text{global}}(\bar c^{\text{ref}})
$$

```python
override = getattr(self, "global_pool_override", None)
if override is not None:
    global_pool = override.expand_as(global_pool)
global_hidden = self.global_encoder(global_pool)
```

把 `global_pool_override` 设为另一首曲子的池化特征，即可在**保持局部鼓点不变**
的前提下替换全局风格——这是风格交换实验的基础。

## 4. 层次化损失：时间段 × 身体部位

实现见 `model/diffusion.py` 的 `_hierarchical_weights` 与 `p_losses`。

### 4.1 权重场

给定配置 $\{\text{frames}=[a,b),\ \text{joints}=\mathcal J,\ \text{weight}=w\}$，
构造两个乘性权重场，除选中的**时间段且选中的身体部位**取值 $w$ 外，其余处处
为 $1$。令 $w_e = w-1$：

$$
M^{\text{time}}_\tau = \mathbb 1[a\le \tau < b],\qquad
M^{\text{joint}}_j = \mathbb 1[j\in\mathcal J]
$$

对重建损失（通道级）与 FK 损失（关节级）分别得到：

$$
W^{\text{chan}}_{\tau,k} = 1 + w_e\, M^{\text{time}}_\tau\, M^{\text{chan}}_k,
\qquad
W^{\text{joint}}_{\tau,j} = 1 + w_e\, M^{\text{time}}_\tau\, M^{\text{joint}}_j
$$

其中关节 $j$ 对应通道 $[7+6j,13+6j)$；若选中根关节 $0$，其平移与触地通道
$[0,7)$ 也一并选入。

### 4.2 加权到重建与 FK 损失

总损失沿用基线的四项加权和，本方法仅对其中重建项与 FK 项施加上述权重场：

$$
\mathcal L = 0.636\,\mathcal L_{\text{rec}} + 2.964\,\mathcal L_{\text{vel}} + 0.646\,\mathcal L_{\text{fk}} + \lambda_{\text{foot}}\,\mathcal L_{\text{foot}}
$$

$$
\mathcal L_{\text{rec}} = \big\| W^{\text{chan}}\odot (f_\theta(x_t,c,t) - x_0) \big\|,
\qquad
\mathcal L_{\text{fk}} = \big\| W^{\text{joint}}\odot (\mathrm{FK}(\hat x_0) - \mathrm{FK}(x_0)) \big\|
$$

其中 $\mathcal L_{\text{vel}}$ 为速度（一阶差分）损失，$\mathcal L_{\text{foot}}$
为脚滑损失（仅在模型自身预测触地的帧上惩罚脚部速度），
$\lambda_{\text{foot}}=10.942$。FK 表示前向运动学，将关节旋转映射到 3D 关节
位置。

```python
loss = self.loss_fn(model_out, target, reduction="none") * chan_w   # 重建
fk_loss = self.loss_fn(model_xp, target_xp, reduction="none") * joint_w  # FK（3D 位置）
```

### 4.3 命令行接口

```bash
--hierarchical_loss_frames 0,75            # 时间段 [0, 75)
--hierarchical_loss_joints 16,17,18,19,20  # SMPL 关节索引（肩/肘/腕）
--hierarchical_loss_weight 1.5             # 强调系数 w
```

SMPL 关节索引（见 `dataset/masks.py`）：`0` 根，`16/17` 肩，`18/19` 肘，
`20/21` 腕，下肢 `1,2,4,5,7,8,10,11`。

## 5. 与 EDGE 编辑能力的关系（非重复性论证）

EDGE 的编辑（`inpaint_loop` + `dataset/masks.py` 掩码）是**采样期的局部硬约束**：
指定关节/帧被强制设为给定真值，其余 inpaint 补全。它需要事先提供目标值，且
没有"全局风格"的概念。

本方法的两个模块与之**层级不同、互补**：

| 维度 | EDGE inpainting | 层次化条件（本文） | 层次化损失（本文） |
|---|---|---|---|
| 作用层级 | 局部（关节/帧） | **全局（整曲风格）** | 训练期区域强调 |
| 作用时机 | 采样期 | 采样期（`global_pool_override`） | 训练期 |
| 约束类型 | 硬替换 | 风格引导 | 软保真度加权 |
| 需要目标动作 | 是 | 否（只需风格向量） | 否 |

即：层次化条件提供了 EDGE **没有的全局控制轴**；层次化损失是**训练先验**而非
运行时硬掩码。

## 6. 评测协议

- 数据：AIST++ 测试集，20 段去重切片；同一变体间使用**相同采样噪声**以保证公平。
- 指标：
  - **PFC**（物理脚部接触）：脚滑，越低越好。
  - **Beat Align**：动作运动学节拍与音乐节拍的对齐度，越高越好。
  - **FID_k**：动力学特征相对真值分布的 Fréchet 距离，越低越好。
  - **Div_k**：动力学多样性，**越接近真值 10.21 越好**（过低=模式坍塌，过高=失真发散）。

### 受控消融主结果（统一 10 epoch 微调、损失权重 1.5）

| 变体 | 全局条件 | 层次损失 | PFC ↓ | Beat Align ↑ | FID_k ↓ | Div_k (→10.21) |
|---|:---:|:---:|---|---|---|---|
| 基线 | – | – | 1.019 | 0.239 | 341.6 | 13.82 |
| + 层次化条件 | ✓ | – | 1.736 | 0.257 | 423.3 | 19.04 |
| + 层次化损失 | – | ✓ | 1.323 | 0.250 | 427.5 | 17.11 |
| **+ 两者（完整方法）** | **✓** | **✓** | **0.619** | **0.264** | **170.6** | 4.91 |

**完整方法在 PFC、Beat Align、FID_k 上均最优**：两个模块单独使用时不稳定，但
组合后取得最低脚滑、最佳节拍对齐，FID 约为基线的一半，体现出明确的协同效应。
